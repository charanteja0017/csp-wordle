Core framework — constraint satisfaction
01

Constraint satisfaction problem

The entire solver is modeled as a CSP. Variables = letter positions (P0..Pn). Domains = possible letters {a-z} at each position. Constraints = rules from green/yellow/grey feedback. Solving = finding an assignment of letters to all positions satisfying every constraint.

Core framework
02

Constraint propagation

Instead of brute-force checking, constraints actively shrink domains. Green fixes a letter. Yellow removes it from one position but ensures it exists elsewhere. Grey removes it globally. Each constraint propagates consequences to all related variables.

Search techniques
03

Backtracking search

Depth-first search that assigns one variable at a time and backtracks when a constraint is violated. Explores the search tree by trying a letter at a position, checking consistency, and undoing (backtracking) if it leads to a dead end. Guarantees completeness.

algo/backtracking.py AIMA Ch. 6.3
Search techniques
04

Informed search / heuristic

The solver doesn't guess blindly. It uses entropy as a heuristic to evaluate which word to guess next — analogous to how A* uses h(n) to pick the most promising node. The cost here is expected remaining candidates after the guess.

algo/entropy.py → rank_guesses() AIMA Ch. 3.5-3.6
Variable & value ordering
05

Minimum remaining values (MRV)

Fail-first strategy. Always pick the variable (letter position) with the fewest remaining domain values. If P3 has only {a, e} while P0 has {a-z}, assign P3 first. Failures surface early, pruning more of the search tree, exponentially fewer nodes explored.

algo/backtracking.py → select_variable_mrv() AIMA Ch. 6.3.1
Variable & value ordering
06

Least constraining value (LCV)

Fail-last for values. When choosing which letter to try at a position, prefer the letter that eliminates the fewest options from other positions' domains. This maximizes the chance that remaining variables can still be assigned without backtracking.

algo/backtracking.py → order_values_lcv() AIMA Ch. 6.3.1
Variable & value ordering
07

Degree heuristic

When two variables have equal MRV, pick the one involved in the most constraints. More constrained positions are harder to satisfy, so resolving them first avoids deep backtracking later.

algo/backtracking.py AIMA Ch. 6.3.1
Pruning & consistency
08

Forward checking

After every assignment, immediately check all connected unassigned variables and remove inconsistent values from their domains. If any domain becomes empty, backtrack NOW — don't wait to discover the failure later. Catches dead ends one step ahead.

algo/forward_check.py AIMA Ch. 6.3.2
Pruning & consistency
09

AC-3 arc consistency

Stronger than forward checking. For every arc (Pi, Pj), ensure every value in Pi's domain has a supporting value in Pj's domain. If not, remove it. When a domain changes, re-queue affected arcs. Reaches a fixpoint where remaining values are pairwise consistent.

algo/ac3.py AIMA Ch. 6.2.2
Pruning & consistency
10

Singleton propagation

When forward checking reduces a domain to exactly 1 letter, treat it as a forced assignment and propagate. This can trigger a chain reaction: fixing P2='r' removes 'r' from P0..P4 — if P4 drops to 1 value, propagate again. Reaches fixpoint.

algo/forward_check.py AIMA Ch. 6.2
Pruning & consistency
11

Domain filtering

The candidate word list is continuously filtered to only words where every letter at every position is still in that position's domain. Impossible words are pruned without ever being explored.

game/words.py → filter_candidates() AIMA Ch. 6.1
Information theory
12

Shannon entropy maximization

For each candidate guess, compute H = -sum(p * log2(p)). A guess that splits candidates into many equally-sized groups has high entropy = high information gain. This is THE key to solving in 3-4 guesses instead of 5-6.

Information theory
13

Pattern space simulation

For each candidate guess, simulate ALL possible feedback outcomes (3^n patterns for n letters). This is a one-level lookahead, similar to minimax without an adversary. The solver imagines every response and picks the guess with the best average outcome.
